## 0:40–1:35 — Realtime interview and code authority

**Visual:** Start Voice. Show Requesting microphone, Listening, Thinking, Speaking, and Saving. Answer several fields and interrupt one model turn. Briefly show the Type alternative.

**Narration:**

> Now OpenAI Realtime conducts a natural, low-latency conversation. It is not GPT-5.6 Sol; it is the voice layer. As I answer, versioned application tools validate and save grounded values atomically, so a duplicate call or interrupted response cannot silently write twice. If the connection drops, VocaForm rebuilds context from server-owned state and resumes at the first unresolved question. Every answer keeps its source and exact spoken wording, and the keyboard path provides the same core journey. I kept consent, memory, validation, application state, and rendering authoritative in code; the models cannot silently save memory or change an answer.

## 1:35–2:05 — Review, verification, and document

**Visual:** Open Review, run the final check, show the passed state, download the verified PDF, and open it at a readable zoom.

**Narration:**

Final export is tied to this exact session version. VocaForm fills a copied AcroForm PDF, reports one hundred percent placement coverage, and confirms that the uploaded source was not changed.
